## 🔑 Уровни доступа

| Значок | Роль | Описание |
|--------|------|----------|
| 👤 | **Пользователь** | Доступ по умолчанию для всех |
| 👑 | **Администратор** | Расширенные права: sudo, произвольные пути |

### Регистрация администратора

**Команда:** `/register <пароль>`

**Использование:**
```
/register ChangeMeInProduction123!
```

**Что даёт регистрация:**
- ✅ Выполнение команд с `sudo` (для действий из `SudoAllowedActions`)
- ✅ Доступ к произвольным путям в системе (`allow_any_path=true`)
- ✅ Управление сервисами и процессами

**Проверка роли:**
```
/whoami
```
Показывает: "👑 Администратор" или "👤 Пользователь"

### 🔐 Безопасность

#### `/register <пароль>`
Регистрирует вас как администратора для получения расширенных прав.

**Использование:**
```
/register ChangeMeInProduction123!
```

**Что даёт:**
- ✅ Выполнение команд с `sudo`
- ✅ Доступ к произвольным путям в системе
- ✅ Управление сервисами и процессами

**Ответы:**
- `✅ Вы успешно зарегистрированы как администратор!` — успех
- `⚠️ Вы уже являетесь администратором.` — уже зарегистрированы
- `❌ Неверный пароль администратора.` — неправильный пароль

---

#### `/restart`
Перезапускает бота. Доступно только администраторам.

**Использование:**
```
/restart
```

**Требования:**
- 👑 Роль администратора (получить через `/register`)

**Что делает:**
1. Пытается выполнить `systemctl restart agentbot` (Linux systemd)
2. Если systemctl недоступен — завершает процесс (Docker/systemd перезапустит)

**Ответ:**
```
🔄 Перезагрузка бота...

Бот будет перезапущен в течение нескольких секунд.
Если бот не запустился автоматически — проверьте логи.
```

### 📚 Алиасы (База знаний)

Алиасы позволяют создавать персональные сокращения для команд и терминов.

---

#### `/alias <имя> <значение> [тип]`
Создаёт новый алиас.

**Использование:**
```
/alias погода /weather
/alias заметки /note
/alias cymmes это приложение blazertool knowledge
```

**Аргументы:**

| Аргумент | Тип | Обязателен | Описание |
|----------|-----|-----------|----------|
| `имя` | строка | ✅ Да | Название алиаса |
| `значение` | строка | ✅ Да | Команда или описание |
| `тип` | строка | ⬜ Нет | `command` (по умолчанию) или `knowledge` |

**Типы алиасов:**
- **command** — алиас команды (например, "погода" → "/weather")
- **knowledge** — алиас знания для ИИ (например, "cymmes" — "приложение blazortool")

**Пример ответа:**
```
🔹 Алиас создан:
"погода" → "/weather"
```

---

#### `/deletealias <имя>`
Удаляет алиас по имени.

**Использование:**
```
/deletealias погода
```

---

#### `/listaliases`
Показывает все ваши алиасы.

**Использование:**
```
/listaliases
```

**Пример ответа:**
```
📚 Ваши алиасы:

🔹 Команды:
  погода → /weather
  заметки → /note

📖 Знания:
  cymmes — это приложение blazortool
```

### ⏰ Cron-задачи

Планирование задач по расписанию с выполнением через ИИ-агент.

---

#### `/cron <название> <cron-expression> <описание>`
Создаёт новую Cron-задачу.

**Использование:**
```
/cron morning "0 8 * * *" Отправить доброе утро
/cron check_logs "*/30 * * * *" Проверить логи на ошибки
```

**Аргументы:**

| Аргумент | Тип | Обязателен | Описание |
|----------|-----|-----------|----------|
| `название` | строка | ✅ Да | Краткое имя задачи |
| `cron-expression` | строка | ✅ Да | Расписание в формате cron |
| `описание` | строка | ✅ Да | Текстовое задание для ИИ |

**Формат cron-выражения (5 полей):**
```
минута час день месяц день_недели
```

**Примеры:**
| Выражение | Описание |
|-----------|----------|
| `0 10 * * *` | Каждый день в 10:00 |
| `*/5 * * * *` | Каждые 5 минут |
| `0 9 * * 1` | Каждый понедельник в 9:00 |
| `0 0 1 * *` | 1-го числа каждого месяца в 00:00 |
| `30 14 * * 1-5` | В будни в 14:30 |

**Пример ответа:**
```
⏰ Задача создана:
📌 Название: morning
⏱ Расписание: 0 8 * * *
📝 Описание: Отправить доброе утро
📅 Следующее выполнение: 2026-03-02 08:00:00 UTC
```

---

### ⏱ Отложенные задачи (systemd-run)

Управление одноразовыми задачами, которые должны выполниться через определенное время.

#### `/run <задержка> ["описание"] <команда>`
Запускает задачу через указанное время.

**Использование:**
```
/run 15min "Бэкап фото" /home/user/backup.sh
/run 2h "Очистка темпа" rm -rf /tmp/*
/run 45s "Тест" echo "Hello"
```

**Аргументы:**
- `задержка` — время ожидания (например, `15min`, `2h`, `45s`, `30`).
- `описание` — (опционально) описание задачи в кавычках.
- `команда` — команда Linux для выполнения.

---

#### `/timers`
Показывает список всех активных и прошедших таймеров (systemd timers).

**Использование:**
```
/timers
```

---

#### `/stoprun <unit>`
Останавливает отложенную задачу по имени юнита.

**Использование:**
```
/stoprun run-u123
```

## 🤖 Работа с ИИ-агентом

Любое сообщение, **не начинающееся с `/`**, отправляется напрямую в ИИ-агент (Gemini).

**Примеры:**
```
Как установить nginx на Ubuntu?
Напиши SQL-запрос для выборки топ-10 пользователей
Объясни разницу между TCP и UDP
```

### Персональный контекст

ИИ-агент использует ваш персональный контекст:
- **Алиасы команд** — если вы написали "погода Москва", ИИ поймёт это как "/weather Москва"
- **Алиасы знаний** — если вы создали алиас "cymmes — приложение blazortool", ИИ будет использовать это знание в ответах
- **История чата** — ИИ помнит контекст разговора (до 20 последних сообщений)

```json
{
  "Bots": {
    "Telegram": {
      "ApiToken": "ВАШ_TELEGRAM_BOT_TOKEN"
    }
  },
  "AiAgent": {
    "ApiKey": "ВАШ_GEMINI_API_KEY",
    "Model": "gemini-1.5-pro"
  },
  "WeatherApiKey": "ВАШ_OPENWEATHERMAP_KEY",
  "AppBaseDir": "/app",
  "Memory": {
    "MaxMessagesPerChat": "20",
    "DatabasePath": "conversations.db"
  },
  "Alias": {
    "DatabasePath": "aliases.db"
  },
  "Cron": {
    "DatabasePath": "cron.db"
  },
  "Keyboard": {
    "DatabasePath": "keyboard.db"
  },
  "LinuxCMD": {
    "AllowSudo": false,
    "AllowedDirs": "logs,scripts,data",
    "AllowedActions": "view_log,service_status,run_script",
    "SudoAllowedActions": "service_status,run_script"
  }
}
```

## ❓ FAQ

**Q: Бот не отвечает на мои сообщения**
A: Проверьте, что `Bots:Telegram:ApiToken` заполнен. Проверьте логи.

**Q: `/weather` говорит "инструмент не настроен"**
A: Заполните `WeatherApiKey` в `appsettings.json`.

**Q: Как работает система алиасов?**
A: Алиасы хранятся в SQLite и автоматически подставляются в контекст ИИ-агента. Если вы создали алиас "погода" → "/weather", то при получении сообщения "погода Москва" оно будет преобразовано в "/weather Москва".

**Q: Как создать задачу по расписанию?**
A: Используйте команду `/cron <название> "<cron>" <описание>`. Задача будет выполняться автоматически, а результат отправляться вам в чат.

**Q: Как стать администратором?**
A: Используйте команду `/register <пароль>`. Пароль задаётся в `appsettings.json` → `Security:AdminPassword`. После регистрации вы получите расширенные права: выполнение команд с `sudo` и доступ к произвольным путям.

**Q: Как проверить свою роль?**
A: Используйте команду `/whoami` — она покажет "👑 Администратор" или "👤 Пользователь".

**Q: Как ИИ узнаёт о моих правах?**
A: ИИ-агент видит ваш chatId и может проверить права через команду `/whoami`. Для выполнения команд с `sudo` или доступа к системным путям (/etc, /var/log) ИИ должен передать параметры `use_sudo=true` или `allow_any_path=true` в инструменте LinuxCMD.

**Q: Можно ли использовать sudo в Linux-командах?**
A: Да, но только для администраторов. Настройте `LinuxCMD:AllowSudo: true` и укажите действия в `SudoAllowedActions`. Обычные пользователи не могут выполнять команды с `sudo`.

**Q: Почему ИИ не использует allow_any_path?**
A: ИИ-агент должен проверять вашу роль через `/whoami` перед использованием `allow_any_path` или `use_sudo`. Если вы администратор — явно попросите ИИ: "выполни с правами администратора" или "используй allow_any_path".

**Q: Как сбросить историю разговора?**
A: Удалите файл `conversations.db` и перезапустите бота.
