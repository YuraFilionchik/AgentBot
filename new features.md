1. Alias/Knowledge base.
    Хранение и использование alias для команд пользователя и для терминов и названий, которые могут быть использованы в подсказках для LLM. 
    Например, пользователь может задать alias "погода" для команды "/weather", "заметки" для команды "/note" и т.д. Или "cymmes" - "это название приложения с именем процесса blazortool"
    Команды должны храниться в базе данных и быть доступны только зарегистрированным пользователям. 
    Также необходимо добавить команду "/alias" для управления alias'ами.
    Пример использования:
    /alias погода /weather  
    /alias заметки /note
    /alias погода Москва

    для удаления alias'а используется команда /delete_alias <alias>
    Применение alias'ов:
    При получении сообщения от пользователя и перед отправкой  в LLM формируется список alies для ИИ и формируется соответствующий запрос к ИИ с этой информацией, чтобы он мог понять команду использовать их в своих ответах. Например, если пользователь отправил "погода Москва", то ИИ должен понять, что это команда "/weather" с аргументом "Москва". Или "cymmes" - "это название приложения с именем процесса blazortool" - это название приложения, которое нужно запустить или проверить статус, в зависимости от контекста.

    Alias - база знаний для ИИ. 

2. Подготовить обертку для LLM запроса от пользователя. 
    В обертку передается информация о пользователе, его роли, alias'ы, список доступных команд и их описание. 
    Также передается история переписки с пользователем. 
    Обертка формирует запрос к LLM и возвращает ответ. 
    Сформироввать структуру ответа в соответствии с архитектурой приложения.
    
3. Предусмотреть возможность отправки обратно пользователю файлов. 
    Например, если пользователь запросил информацию, в которой много данных - LLM может принять решение о необходимости сохранения этой информации в файл и отправить его пользователю. 
    Также предусмотреть возможность отправки файлов пользователю по команде. Например, если пользователь запросил список файлов в папке, то бот должен отправить простым сообщением список файлов в чате пользователю. 

4. cron задачи. 
    Возможность планирования задач по расписанию. 
    Задача - это просто текстовое описание того, что нужно сделать для LLM 
    Например задача может быть такой: "Проверить статус приложения cymmes и если оно не запущено, то запустить его". 
    Или "Отправить сообщение пользователю в 10:00 с текстом: 'Не забудьте выпить таблетки'"
    Или "Выполни чтение логов cymmes и если в логах есть ошибка 'error', то отправить сообщение пользователю с текстом: 'В логах cymmes найдена ошибка'"
    Добавить команды редактирования, удаления, просмотра, добавления cron задач. 
    
5. Кнопки в чате Telegram.
    Спроектировать и сформировать структуру ReplyKeyboardMarkup` (обычная клавиатура) и InlineKeyboardMarkup` (inline-клавиатура) для команд бота. ReplyKeyboardMarkup для часто используемых команд, InlineKeyboardMarkup для команд, которые используются редко или для расширения функциональности ReplyKeyboardMarkup кнопок, например расширенное меню.
    В ReplyKeyboardMarkup должна быть кнопка "Quick commands", которые загружаются из базы данных и могут меняться самим пользователем по запросу к ИИ (предусмотреть возможность добавления, удаления, изменения через дополнительные tools методы).
